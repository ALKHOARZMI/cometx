# CometX - دليل البدء السريع

## المتطلبات الأساسية

قبل البدء، تأكد من تثبيت ما يلي:

- **Node.js** 20.x أو أحدث
- **npm** 10.x أو أحدث
- **Git** لاستنساخ المستودع

## التثبيت

### 1. استنساخ المستودع

```bash
git clone https://github.com/ALKHOARZMI/cometx.git
cd cometx
```

### 2. تثبيت التبعيات

```bash
npm install
```

سيؤدي هذا إلى تثبيت جميع التبعيات المطلوبة بما في ذلك:
- React 18 و React DOM
- TypeScript
- أداة البناء Vite
- Tailwind CSS
- Chart.js
- @xenova/transformers
- محرر Monaco
- XTerm.js
- i18next للترجمة والتدويل

### 3. بدء خادم التطوير

```bash
npm run dev
```

سيبدأ التطبيق على `http://localhost:5173` (أو منفذ آخر إذا كان 5173 مشغولاً).

## نظرة عامة على الميزات

### لوحة التحكم
- عرض مقاييس النظام في الوقت الفعلي
- رسوم بيانية تفاعلية تظهر استخدام المعالج والذاكرة
- مؤشرات حالة النظام

### المحادثة الذكية
- معالجة ذكاء اصطناعي محلية 100%
- واجهة محادثة تحترم الخصوصية
- لا يوجد اعتماد على السحابة
- مدعوم بـ Transformers.js

### الطرفية
- محاكي طرفية كامل الميزات
- أوامر مدمجة (help, clear, echo, date, whoami, about)
- تكامل احترافي مع xterm.js

### محرر الأكواد
- محرر Monaco (محرر VS Code)
- تمييز بناء الجملة للعديد من اللغات
- تنفيذ الأكواد (JavaScript فقط للأمان)
- حفظ الأكواد في ملفات محلية

### مصنع التطبيقات
- إنشاء تطبيقات جديدة من القوالب
- قوالب مشاريع متعددة متاحة
- نظام إنشاء سريع

### المنسق
- إدارة خدمات النظام
- بدء وإيقاف وإعادة تشغيل الخدمات
- مراقبة حالة الخدمة واستخدام الموارد

## البناء للإنتاج

### إنشاء بناء الإنتاج

```bash
npm run build
```

ستكون الملفات المبنية في دليل `dist/`.

### معاينة بناء الإنتاج

```bash
npm run preview
```

## دعم اللغات

يدعم CometX كلاً من الإنجليزية والعربية مع دعم RTL (من اليمين إلى اليسار):

1. انقر على أيقونة اللغة في الشريط الجانبي
2. اختر "العربية" للعربية أو "English" للإنجليزية
3. ستقوم الواجهة تلقائيًا بضبط اتجاه التخطيط

## التخصيص

### تغيير ألوان السمة

يمكن تخصيص سمة النيون الخضراء في `src/index.css`. ابحث عن قيم الألوان:
- اللون الأخضر الأساسي: `#10b981`
- اللون الأخضر المميز: `#4ade80`
- الخلفية الداكنة: `#030712`

### إضافة صفحات جديدة

1. أنشئ مكونًا جديدًا في `src/pages/`
2. استورده في `src/App.tsx`
3. أضف عنصر التنقل في `src/components/Layout.tsx`
4. أضف الترجمات في `src/locales/en.json` و `src/locales/ar.json`

## نصائح التطوير

### استبدال الوحدة الساخنة (HMR)

يوفر Vite HMR فوريًا. ستظهر تغييراتك على الفور بدون تحديث الصفحة.

### فحص نوع TypeScript

```bash
npm run build
```

يقوم هذا الأمر بتشغيل مترجم TypeScript قبل البناء، للكشف عن أخطاء النوع.

### Linting

```bash
npm run lint
```

قم بتشغيل ESLint للتحقق من جودة الكود.

## استكشاف الأخطاء وإصلاحها

### المنفذ قيد الاستخدام بالفعل

إذا كان المنفذ 5173 مشغولاً، سيستخدم Vite تلقائيًا المنفذ التالي المتاح. تحقق من إخراج وحدة التحكم للمنفذ الفعلي.

### أخطاء البناء

إذا واجهت أخطاء في البناء:

1. احذف `node_modules/` و `package-lock.json`
2. قم بتشغيل `npm install` مرة أخرى
3. امسح ذاكرة التخزين المؤقت للمتصفح
4. حاول `npm run build` مرة أخرى

### مشاكل تحميل نموذج الذكاء الاصطناعي

يتم تنزيل نموذج الذكاء الاصطناعي عند الاستخدام الأول. تأكد من أن لديك:
- اتصال إنترنت مستقر (المرة الأولى فقط)
- مساحة قرص كافية (~500 ميجابايت)
- متصفح حديث مع دعم WebAssembly

### مشاكل الأداء

لأداء أفضل:
- استخدم متصفحًا قائمًا على Chromium (Chrome، Edge، Brave)
- تأكد من تمكين WebGL
- أغلق علامات تبويب المتصفح غير الضرورية
- اسمح بذاكرة وصول عشوائي كافية (4 جيجابايت كحد أدنى موصى به)
